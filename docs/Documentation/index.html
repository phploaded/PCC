PCC - Project Cost Calculator

Updated: 2026-02-27

This version runs on jQuery 4.0.0 and includes a 10-scenario real-life demo hub, schema-driven themed demos, strict validation, quote persistence, local autosave/share state, and print/PDF/CSV export from live summary.

1) Package Structure

  • index.php - scenario hub listing 10 real-life demo types
  • scenario-demo.php - runtime page for each scenario (?scenario=slug)
  • scenario-demo.js - schema configs + runtime logic for all scenarios
  • scenario-demo.css - themed styling variants per scenario
  • builder.php - visual schema builder + live renderer
  • legacy-demo.html - retained legacy demo
  • demo.css - previous single-demo stylesheet (retained)
  • builder.css - visual builder page styling
  • lib-pcc/jquery-4.0.0.min.js - jQuery runtime
  • lib-pcc/jquery.pcc.js - plugin logic
  • lib-pcc/jquery.pcc.min.js - synchronized plugin build
  • lib-pcc/pcc.css - plugin UI styles (summary/actions/wizard/modal)
  • send_email.php - secured email endpoint
  • quote_storage.php - SQLite quote storage layer
  • quote_api.php - save/load quote JSON API
  • admin_quotes.php - quote records admin viewer
  • CHANGELOG.md - plugin release history
  • MIGRATION_GUIDE.md - upgrade notes for integrators

2) Base Integration

<script>wpcc_domain = 'https://yourdomain.com/PCC';</script>
<script>wpcc_csrf_token = 'server-generated-token';</script>
<script src="lib-pcc/jquery-4.0.0.min.js"></script>
<script src="https://cdnjs.cloudflare.com/ajax/libs/jspdf/2.5.1/jspdf.umd.min.js"></script>
<script src="lib-pcc/jquery.pcc.js"></script>
<link rel="stylesheet" href="lib-pcc/pcc.css" />
<script>jQuery(function($){ $('#my-form').pcc(); });</script>

If jsPDF is not loaded, PDF action falls back to print mode.

3) Required Form Markup

Each calculator form must use class pcc-form with an ID. The summary container must be that ID plus -show.

<form id="myquote" class="pcc-form" data-curr=" USD">...</form>
<div id="myquote-show" class="wpcc-preview"></div>

Cost fields

  • data-title - line item label
  • data-cost - fixed cost or this
  • data-mult - multiplier
  • form-level pricing: data-tax-rate, data-discount-fixed, data-discount-percent, data-coupons JSON

4) Strong Validation System

PCC supports required fields, numeric bounds, length bounds, regex, type checks, and custom messages per field.

Supported validation attributes

  • data-required="1"
  • data-min, data-max (numeric bounds)
  • data-minlen, data-maxlen
  • data-pattern (JavaScript regex body)
  • data-validate="email|number"

Custom message attributes

  • data-msg-required, data-msg-min, data-msg-max
  • data-msg-minlen, data-msg-maxlen
  • data-msg-pattern, data-msg-email, data-msg-number
<input
  name="client_phone"
  data-title="Phone Number"
  data-required="1"
  data-pattern="^[0-9+()\\-\\s]{10,20}$"
  data-msg-required="Phone number is required."
  data-msg-pattern="Use a valid phone format."
/>

Invalid fields/groups get .pcc-invalid and are blocked before wizard next step or email send.

5) Multi-step Wizard Mode

  • Enable with data-wizard="1" on form.
  • Wrap each step in .pcc-step and optional data-step-title.
  • Plugin adds progress bar, step labels, and previous/next navigation.
  • Validation runs per current step when pressing Next.
<form class="pcc-form" id="quote" data-wizard="1">
  <section class="pcc-step" data-step-title="Scope">...</section>
  <section class="pcc-step" data-step-title="Contact">...</section>
</form>

6) Save/Restore + Share State

  • Autosave draft to LocalStorage on form changes.
  • Save Quote writes quote to SQLite via quote_api.php?action=save.
  • Load Last restores most recently saved quote for this form.
  • Share copies URL with pcc_state payload for browser-to-browser restore.
  • Saved quote IDs are also reflected using URL param quote_id.

7) Export Options

  • Print - opens printable quote view.
  • CSV - exports summary lines as CSV file.
  • PDF - generates PDF using jsPDF if available.

All exports are generated from the live summary output (#formid-show).

8) Conditional Logic + Coupons

  • data-show-if supports field=value, field!=value, OR with |, and multiple clauses with ;.
  • Hidden conditional blocks are disabled and excluded from pricing/validation.
  • Coupon rules come from form data-coupons JSON map.
Example: data-show-if="timeline=2500;project_scope!=8500"

9) jQuery Plugin API

PCC now exposes a public API for host projects:

$('#my-form').pcc(); // init
$('#my-form').pcc('recalc');
$('#my-form').pcc('validate');
$('#my-form').pcc('getState');
$('#my-form').pcc('setState', stateObj);
$('#my-form').pcc('goToStep', 2);
$('#my-form').pcc('undo');
$('#my-form').pcc('redo');
$('#my-form').pcc('destroy');

Utility object: $.pcc.version, $.pcc.defaults, $.pcc.registerAdapter(name, binder), $.pcc.recalcAll().

10) Events and Hooks

Plugin emits extension events for host apps and no-core-edit customization.

  • pcc:beforeCalc
  • pcc:afterCalc
  • pcc:validationFail
  • pcc:stepChange
  • pcc:stateChange
$('#my-form').on('pcc:afterCalc', function (e, payload) {
  console.log(payload.finalTotal);
});

11) Formula Engine and Advanced Rules

  • Use data-formula with field references: {field_name}.
  • Supported math helpers: min, max, abs, round, ceil, floor, pow.
  • Conditional rules now support &&, ||, (), and comparisons >, <, >=, <=, !=, =.
<input data-formula="({floor_area}*0.3)+({room_count}*120)" />
<div data-show-if="(timeline=2500 || project_scope=22000) && floor_area>=1000"></div>

12) Schema Mode and Visual Builder

  • Use $.pcc.validateSchema(schema) to validate config.
  • Use $.pcc.schemaToHtml(schema) to generate markup string.
  • Use $.pcc.renderSchema(target, schema, options) to render and initialize live form.
  • Builder page builder.php provides visual field/step editing and JSON transport workflow.
const result = $.pcc.renderSchema('#preview', schemaConfig, {
  skipRestore: true,
  autosave: false
});

Builder supports copy JSON, download JSON (Blob API), import JSON, and render back to interactive form.

13) Rule Debugger Panel

  • Enable globally with option ruleDebugger: true or form attribute data-rule-debugger="1".
  • Shows real-time formula expansion/evaluation and conditional rule pass/fail outputs.
  • Useful when building and testing complex schema rules.

14) Repeaters, Adapters, and History

  • Repeaters: data-pcc-repeater + data-pcc-repeat-item with built-in add/remove actions.
  • Input adapters: built-in hooks for Select2, datepicker, slider, and masked inputs.
  • State history: undo/redo buttons and keyboard shortcuts (Ctrl/Cmd+Z, Ctrl/Cmd+Y, Shift+Ctrl/Cmd+Z).

15) i18n, Theming, Templates, Accessibility

  • Locale-aware number formatting via data-locale / options.
  • Theme variables via form options (theme map -> --pcc-* CSS vars).
  • Template callbacks for pricing rows, action buttons, quote meta, and validation output.
  • Accessibility upgrades: aria-live totals, dialog semantics, keyboard wizard navigation, and error associations.

16) Quote API and Admin

  • POST quote_api.php?action=save with CSRF token stores quote payload.
  • GET quote_api.php?action=get&quote_id=... returns quote JSON.
  • admin_quotes.php lists saved quotes and shows quote detail.

SQLite database file is auto-created at data/pcc_quotes.sqlite.

17) Security Notes

  • send_email.php validates CSRF, honeypot, and request timing.
  • Rate limiting is enforced (5 requests / 15 minutes).
  • Input is sanitized before email generation.
